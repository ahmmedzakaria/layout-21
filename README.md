# Sentinel KYC — Angular Conversion

Angular port of the KYC compliance console POC, built to the finalized scope:

- Angular **21 LTS**, standalone components throughout (no NgModules)
- TypeScript, strict compiler + strict template checks (see `tsconfig.json`)
- **Zoneless change detection**; **Signals** for local/UI state, **RxJS** reserved for async streams (HTTP, etc.)
- **Angular CDK** for behavior primitives (Overlay/positioning) — no Angular Material component layer, no Bootstrap
- Custom **SCSS design-token system**, carried over 1:1 from the original CSS custom properties — all 7 themes preserved
- Custom **SVG icon set**, ported as an Angular icon component + registry — no Font Awesome
- **i18n** via `@jsverse/transloco`, with `DirectionService` handling RTL for Arabic
- **Vitest** for unit tests, **Playwright** for end-to-end
- `angular-eslint` + Prettier for code quality

## Getting started

This was hand-authored to the target file layout rather than generated by `ng new`,
since dependency resolution requires network/npm access. Before running it:

```bash
npm install
npm start        # ng serve on http://localhost:5200
npm test         # Vitest unit tests
npm run test:e2e # Playwright e2e (starts the dev server automatically)
npm run lint
```

`package.json` pins dependency ranges (`^21.0.0`, etc.) rather than exact patch
versions — run `npm install` once to let npm resolve and lock the exact versions
current at install time, then commit the resulting `package-lock.json`.

## How each piece of the original vanilla-JS POC maps over

| POC mechanism | Angular equivalent |
|---|---|
| `manualExpanded` / `openModule` globals | `RailStateService` (signals) |
| `toggleDD()` / `closeAllDD()` / manual `document` click listener | `HeaderMenuService` + `HeaderDropdownComponent` (CDK Overlay handles backdrop/dismiss) |
| `positionFlyout()` (manual `getBoundingClientRect` math) | `RailFlyoutService` + CDK `cdkConnectedOverlay` (`ConnectedPosition`) |
| `THEMES` object + `selectTheme()` + `body.dataset.theme` | `ThemeService` (signal + `effect()` syncing to `<body>`) |
| `:root` / `body.dark` / `body[data-theme]` CSS custom properties | `src/styles/_tokens.scss` — same variable names, same values |
| Raw SVG strings inlined per element | `IconComponent` + `icon-registry.ts` |
| Hardcoded `EN`/`BN`/`AR` strings | `@jsverse/transloco` + `assets/i18n/*.json` + `DirectionService` for RTL |
| `mods` array (module → category → items) | `NAV_MODULES` in `core/models/nav-module.model.ts` |

## What's intentionally left as a stub

- **Auth/session**: `logout()` in `HeaderComponent` logs to console where a real
  app would clear the session and redirect to the IdP.
- **SSO app launch**: `launchApp()` likewise stubs the redirect — wire to your
  SSO broker's signed-URL endpoint.
- **Customers / KYC / Admin feature routes**: only `Dashboard` is wired up in
  `app.routes.ts`; the other three modules' routes follow the same
  `loadComponent` pattern once those screens are built.
- **Real notification/task data**: currently static markup in the header
  dropdowns — swap for a service + `HttpClient`/RxJS stream when the API exists.
